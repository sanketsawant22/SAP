# 📦 What is O2C (Order to Cash)?

Imagine you own a company called **TechMart**.

A customer wants to buy a laptop.

What happens?

```
Customer asks about product
        ↓
Inquiry
        ↓
Quotation
        ↓
Sales Order
        ↓
Delivery
        ↓
Billing (Invoice)
        ↓
Customer Pays
```

That's called the **Order to Cash Process (O2C)**.

SAP simply records every step.

---

# Step 1️⃣ Inquiry

### T-Code

```
VA11 Create
VA12 Change
VA13 Display
```

Document Type

```
IN
```

Think of this like:

> Customer:
> "I want 100 laptops.
> Can you tell me price and availability?"

No order yet.

No payment.

No stock movement.

Just asking.

Example

```
ABC Company

↓

Needs 100 laptops

↓

Creates Inquiry
```

SAP stores this information.

---

# Step 2️⃣ Quotation

### T-Code

```
VA21 Create
VA22 Change
VA23 Display
```

Document Type

```
QT
```

Now your company replies.

Example

```
100 laptops

₹50,000 each

Delivery in 5 days

Offer valid for 15 days
```

That's a quotation.

Still...

❌ No stock reserved

❌ No billing

Only an offer.

This represents the legal company.

# What is PGI?

This is a very important concept.

Before PGI

```
Company owns goods.
```

After PGI

```
Goods have left warehouse.

Inventory decreases.

Customer now owns goods.
```

PGI changes stock.

Without PGI,

Delivery isn't completed.

# 💰 Pricing Procedure (Preview)

Your trainer intentionally skipped pricing because it's a separate topic.

You noted:

```
VOKK
```

Pricing depends on:

* Sales Area
* Document Pricing Procedure
* Customer Pricing Procedure (from Customer Master)
* Pricing Procedure assigned (e.g., `RVAA01`)

Think of `VOKK` as the place where SAP decides **which pricing procedure to use** for a particular combination of sales area, customer, and document type.

We'll cover this in detail later.
